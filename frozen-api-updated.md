# Frozen API — Mythras Imperative

Signatures on this page are called directly by external consumers (the
planned node editor, modules, macros). **Do not change a frozen signature
without updating this doc and the runtime in the same change.** A signature
change here is a breaking change for every caller listed.

This file did not exist before v1.4.250 — it is being initialized with the
entries verified as part of the HP-max lock work. It is not yet a complete
audit of every frozen surface in the codebase; expand it as other
already-frozen entry points are confirmed.

| Symbol | Signature | Caller(s) | Notes |
|---|---|---|---|
| `SE_RESOLVERS` | `{ [seId: string]: (ctx) => Promise }` | Node editor (planned), `CombatEngine._resolveOpposedSEs`, `_afterDefenceResolved` | Exported from `module/combat/effects/index.js`. Keys match `id` in `CONFIG.MYTHRAS.specialEffects`. |
| `game.system.api.syncHitLocationHP` | `(actor: Actor) => Promise<void>` | Destined module, macros, GM console | Sole writer of hit-location item `system.hp` (max). Attached in the `ready` hook via `Object.freeze({ syncHitLocationHP })`. Also exported from `mythras.mjs` for internal use by the `updateActor` hook. See `extension-point-api-updated.md` for the `hitPointBonusHooks` contract it consumes. Idempotent — safe to call repeatedly; only writes locations whose computed max differs from the stored value. |
| `game.system.api.determineOutcome` | `(result: number, target: number, rawSkill?: number) => 'critical'\|'success'\|'failure'\|'fumble'` | Destined module (`module/utils/roll-gating.js`), planned node editor | **The canonical unopposed skill grader.** Exported from `module/utils/roll-math.js`; also the function `CombatEngine`/`CharacterSheet` itself uses internally for every skill and combat roll, so a module calling this is guaranteed to match engine grading exactly, not a lookalike. `result` is the rolled d100; `target` is the effective skill after difficulty; `rawSkill` (defaults to `target`) is the pre-difficulty skill, consulted only for the fumble threshold (99 fumbles when `rawSkill < 100`; 100 always fumbles regardless). **This is the intended path for modules building roll-gated mechanics — call this rather than reimplementing crit/fumble bands independently.** Pure, no side effects, safe to call from anywhere once `ready` has fired. `module/utils/combat-math.js` has a second, functionally-identical copy of this function used internally by opposed resolution — that copy is **not** exposed here; `roll-math.js` is the canonical, documented home. **Not yet deduplicated system-side** — no dedicated known-issues tracking doc exists in this repo to log that against (the module repo has `PHASE4-STATE.md`'s "On the Horizon" for this purpose; this repo has no equivalent), so it's recorded here instead until one exists. |
| `game.system.api.shiftGrade` | `(gradeId: string, steps: number) => string` | Destined module (boosts whose effect scales with success grade) | Shifts a difficulty-grade id along `GRADE_ORDER` by `steps` (negative = easier, positive = harder), clamped to the ends of the list. Grade-scaling primitive, not an outcome grader — pairs with `determineOutcome`/`GRADE_ORDER`/`applyDifficulty` as one toolkit, all from `roll-math.js`. |
| `game.system.api.GRADE_ORDER` | `string[]` (7 entries: `veryEasy`…`hopeless`) | Destined module, `shiftGrade`'s own consumers | The canonical ordered list of difficulty-grade ids `shiftGrade` walks along. Frozen array reference (the object itself is not deep-frozen, but nothing in the system mutates it and callers must not either). |
| `game.system.api.applyDifficulty` | `(skill: number, difficulty: string) => number` | Destined module (boosts that roll at a stated difficulty grade) | Applies a difficulty-grade multiplier to a raw skill value, per `DIFFICULTY_GRADES`. Returns the skill unchanged if `difficulty` doesn't match a known grade or is `'hopeless'` (multiplier `null`, meaning "cannot be attempted," not "multiply by null"). |
| `game.system.api.DIFFICULTY_GRADES` | `{ [gradeId: string]: { multiplier: number\|null } }` | Destined module (callers of `applyDifficulty` wanting the raw table) | The multiplier table `applyDifficulty` reads. Exposed alongside `applyDifficulty` specifically so a caller doesn't have to guess the multiplier for a grade it wants to display or validate against. |

## Change log

- **v1.4.260** — `game.system.api` extended with `determineOutcome`/`shiftGrade`/`GRADE_ORDER`/`applyDifficulty`/`DIFFICULTY_GRADES` (all from `module/utils/roll-math.js`), promoting the effect-application layer's roll-gating primitive off a fragile module-side cross-package relative import (Destined's `roll-gating-prompt-v2.md` batch, module v1.9.54) onto the documented frozen-API contract. Pure functions and a frozen constant — no behavior change to anything that existed before; purely additive to the `game.system.api` object. `combat-math.js`'s duplicate `determineOutcome` and the opposed helpers (`resolveOpposedRoll`/`resolveDifferential`) are deliberately **not** exposed in this batch — see the table notes above.
- **v1.4.250** — `game.system.api.syncHitLocationHP` added as the first frozen `game.system.api` entry.
